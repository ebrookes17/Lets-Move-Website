2.4.6 JavaScript

a) Use of variables, arithmetic, comparison and logical operators, subroutines, functions, arrays, conditional statements, conditional operators, running time commands with system clock.

These features can be found in "lastModifiedDate.js", "formCountDown.js", and "validateForm.js".

b) Use of system date (example: clock, countdown timer to an event) on your web site.

sign-up.html features a 20-minute countdown clock. The form must be filled out and submitted before the timer expires, or the user will be notified through an alert (created with JavaScript) and they will be returned to the home page. The JavaScript file is "formCountDown.js".

c) Validating form input by using appropriate JavaScript functions.

The form on sign-up.html mainly makes use of the "required" feature of HTML to check if a field has been entered or not, but for the question "*Please select which rate applies to your purchase: " which uses the "radio" input type, the JavaScript file "validateForm.js" will check if this question has been answered. If no selection has been made, the user will be prompted through an alert that the question must be answered. If it has been answered, and every other field has been filled out, it will notify the user that the form has been successfully submitted and return them to the home page. Note that the HTML form feature "required" will be checked FIRST, so the alert will not come up if previous fields have not been answered first. Instead, the "this field is required" message will come up, until those fields have been answered.

d) Use of JavaScript alerts for validation purposes.

sign-up.html features the alert created in JavaScript which is described in the paragraph above ( 2.4.6 - c) ). The code can be found in "validateForm.js".

e) Use of BOM/DOM in emulating events through the use of event handlers.

The JavaScript file "formCountDown.js" used on sign-up.html uses event handlers to check certain conditions, like if the form has been submitted before the countdown has expired. "validateForm.js" also uses event handlers to check certain conditions.

2.4.8 Call to Action (CTAs)

Your website must include at least three CTAs, in particular, on the homepage of the website.

index.html features 3 CTAs: a button in the top-right corner of the navbar, a button in the center of the main image, and a large banner-like button at the bottom of the page before the footer. All CTAs encourage the user to sign up at the studio, but are all phrased in different ways. The one at the bottom before the footer mentions that new students receive 20% off.
These same CTAs are implemented on almost every page of the website. In particular, the CTA in the top-right corner of the navbar is present on every page because it makes it easy for the user to sign up at the studio, as this button is at the same location on every page.
